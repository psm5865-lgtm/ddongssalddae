# 똥쌀때 — 배포 & 출시 가이드

> 정적 SPA(빌드 결과가 순수 HTML/JS/CSS)라 서버가 필요 없어요.
> **무료로 올려 시작 → 사용자가 늘면 Vercel Pro로 전환**(코드 변경 0)하는 경로로 준비돼 있어요.

---

## 0. 준비 상태 (이미 해둔 것)

| 항목 | 파일 | 의미 |
|---|---|---|
| SPA 딥링크 라우팅 | `vercel.json` rewrites | `/session` 등으로 새로고침/푸시 진입해도 404 안 남 |
| 에셋 캐시·보안 헤더 | `vercel.json` headers | 정적 파일 1년 캐시 + 안전 헤더. **X-Frame 미설정**(토스 webview 임베드 허용) |
| 파비콘·매니페스트 | `public/favicon.svg`, `public/manifest.webmanifest` | 탭 아이콘·홈화면 추가 |
| 메타/OG | `index.html` | 설명·공유 미리보기 |
| 에러 화면 | `src/components/ErrorBoundary.tsx` | 런타임 에러 시 흰 화면 대신 "처음으로" 화면 |
| 사용량 측정 | `@vercel/analytics` (main.tsx) | 쿠키리스. 배포 후 대시보드에서 켜면 동작 |
| Node 버전 | `.nvmrc` (20), `engines` | 재현 가능한 빌드 |

빌드 설정: **Build `npm run build` / Output `dist`** (Vercel이 Vite로 자동 인식, vercel.json에도 명시).

## 1. Vercel에 무료로 올리기

1. 코드를 **GitHub 저장소**에 올려요. (아직 git이 없다면 아래 "부록 A")
2. [vercel.com](https://vercel.com) 가입 → **Add New → Project → 저장소 Import**
3. Framework = **Vite** 자동 감지 → 그대로 **Deploy**
4. `https://ddongssalddae.vercel.app` 같은 주소가 나와요. 이후 **push 할 때마다 자동 배포**.

CLI로도 가능: `npx vercel`(미리보기) / `npx vercel --prod`(프로덕션).

## 2. 무료(Hobby) vs Pro — 언제 올리나

| | Hobby (무료) | Pro ($20/월·멤버) |
|---|---|---|
| 대역폭 | 약 100GB/월 | 1TB/월~ |
| 용도 약관 | **비상업적·개인용** | 상업적 사용 가능 |
| 분석 | 제한적 | 풍부 |
| 비밀번호 보호 등 | ✕ | ○ |

**중요(약관):** Vercel 무료 플랜은 원칙적으로 **비상업적 용도**예요. 토스에 공개 출시한 무료 앱은 회색지대지만, **광고·결제 등 수익이 붙는 순간 Pro로 올려야** 약관에 맞아요. 업그레이드는 **결제 설정만 바꾸면 끝**이고 `vercel.json`/코드는 그대로예요.

**업그레이드 신호:**
- 월 대역폭이 무료 한도에 근접
- 수익화 시작(광고·인앱결제 등)
- 더 자세한 트래픽 분석/비밀번호 보호가 필요

**무료로 더 버티고 싶다면 대안:** **Cloudflare Pages**(상업적 사용 허용 + 대역폭 사실상 무제한, 정적 SPA에 적합). 설정은 Build `npm run build` / Output `dist` 동일. 단, `vercel.json`은 안 먹으므로 SPA 폴백은 `public/_redirects`에 `/* /index.html 200` 한 줄로 대체. (지금은 Vercel 기준으로 준비돼 있어요.)

## 3. 커스텀 도메인 (선택)

Vercel 프로젝트 → Settings → Domains에서 도메인 추가 후 DNS 연결. 무료/Pro 모두 가능, HTTPS 자동.

---

## 4. 토스 미니앱에 연결

1. 위에서 나온 **배포 URL(https)** 을 확보.
2. 앱인토스 콘솔 → 워크스페이스(검수 중인 앱과 **같은 워크스페이스**) → **미니앱의 URL**로 이 주소 등록.
3. 토스가 이 URL을 webview로 띄워요. (그래서 `vercel.json`에서 프레임 차단 헤더를 빼둠.)
4. 딥링크: 푸시 랜딩 URL을 `https://.../session`으로 주면 세션으로 바로 진입(미온보딩 사용자는 온보딩으로 안내됨).

## 5. 출시 전 체크리스트

- [ ] 실제 기기(iOS/Android)에서 토스 webview로 동작 확인
- [ ] **앱 아이콘 PNG**(512px 권장) 준비 — 콘솔 업로드용. (favicon.svg는 브라우저 탭용)
- [ ] **개인정보처리방침 / 이용약관** URL (localStorage만 쓰고 서버 전송 없음을 명시하면 간단)
- [ ] 미니앱 정보: 서비스명·설명·**고객센터 연락처** (알림 동의문에 자동 반영)
- [ ] 카피·문구 검수 제출 (영업일 2~3일)
- [ ] 빌드 최신화 후 재배포 → 토스 URL 갱신 확인

## 6. 사용량 모니터링 (스케일 판단용)

- 코드에는 `@vercel/analytics`가 이미 연결돼 있어요(쿠키 없음, 개인정보 친화).
- Vercel 대시보드 → 프로젝트 → **Analytics 탭에서 Enable**만 누르면 수집 시작.
- 여기서 방문/페이지뷰 추세를 보고 **Pro 전환 시점**을 판단하세요.

---

## 7. (선택) 오프라인까지 지원하려면

정적 + localStorage라 네트워크가 거의 필요 없어요. 신호 약한 화장실에서도 열리게 하려면 나중에 `vite-plugin-pwa`(autoUpdate)로 서비스워커를 추가할 수 있어요. 지금은 업데이트 staleness 리스크를 피하려 넣지 않았고, 필요할 때 옵트인 권장.
