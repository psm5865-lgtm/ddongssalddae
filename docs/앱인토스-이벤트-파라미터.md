# 똥쌀때 — 앱인토스 이벤트 · 파라미터 설명

> 앱인토스 프레임워크가 **자동 수집**하는 이벤트/파라미터예요(코드로 따로 안 넣음).
> 콘솔의 "설명" 칸에 그대로 적어두기 위한 참고 문서. (`최초 발생일` 2026.06.02 = 출시 빌드 배포일)

## 이벤트 (Events)

### 화면 조회 — 우리 앱 라우트와 1:1
| 이벤트명 | 설명 |
|---|---|
| `/::screen` | 앱 첫 진입(루트) 화면 조회. 홈/온보딩으로 분기되는 입구 |
| `/home::screen` | 홈 화면 조회 |
| `/session::screen` | 마음챙김 세션(활동) 화면 조회 — '시작하기' 후 진입 |
| `/complete::screen` | 활동 완료(도장) 화면 조회 |
| `/journal::screen` | '내가 적은 기록' 화면 조회 |

### 행동 이벤트 — 플랫폼 자동 수집
| 이벤트명 | 설명 |
|---|---|
| `appsintoss_app_visit::impression__enter_appsintoss` | 미니앱 진입(앱 열기) 시 1회 = **방문 시작** |
| `appsintoss_app_visit::impression__stay_active_heartbeat` | 앱을 보는 동안 일정 간격으로 뛰는 활성 체류 신호. **실제 체류시간 측정용** |
| `appsintoss_app_visit__common_module::impression__stay_time` | 방문 종료 시 총 체류 시간 기록 |
| `appsintoss_app_visit__common_module::impression__navigation_bar` | 토스 상단 내비게이션 바 노출 |
| `appsintoss_app_visit__common_module::click__icon_close` | 상단 닫기(X) 클릭 = 미니앱 종료 |
| `appsintoss_app_visit::background__miniapp_scheme_call` | 딥링크(`intoss://…`)로 미니앱이 호출됨 |

## 데이터 볼 때 팁

- **방문/퍼널**: `enter_appsintoss`(방문 수) → `/home → /session → /complete::screen` 순으로 얼마나 떨어지는지(이탈) 확인.
- **체류시간**(`stay_time`): 우리 앱은 **짧게 끝나는 게 정상**(디톡스). 짧다고 나쁜 게 아니라 **재방문(D1)**이 더 중요한 성공 지표.
- **기능 매력도**: `/journal::screen` 조회로 '기록' 기능 사용도 가늠.
- `_immediate`·`_background`·`schema_id`·`top_yn` 류는 플랫폼 내부값 → 설명만 적고 깊게 안 봐도 됨.
- '추정' 표시 항목은 플랫폼 공통 파라미터라 의미를 추정한 것 — 정확한 정의는 앱인토스 콘솔 가이드 참고.
