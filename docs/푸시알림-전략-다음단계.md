# 똥쌀때 — 푸시/알림 전략 & 다음 할 일

> 토스 앱인토스 "스마트 발송" 기준으로 정리. 비용 0원 유지, 같은 워크스페이스 전제.

---

## 0. 결론 한 줄

> **똥쌀때는 알림을 기본 OFF로 두고, 사용자가 직접 켠 "하루 1회 정기 마음챙김 알림"만 보낸다.**
> 발송은 서버 없이 '토스에게 발송 요청' → **추가 비용 0원**.

이게 디톡스 철학(앱이 사람을 다시 끌어들이지 않음)과 토스 검수(불안 조성·리텐션 미끼 반려) 양쪽을 모두 만족하는 유일한 안전 지대예요.

---

## 1. 지금 상황 (같은 워크스페이스 전제)

```
워크스페이스 (사업자 1개)
 ├─ [검수 중인 앱]   ← 지금 영업일 2~3일 대기 중
 └─ [똥쌀때]         ← 같은 워크스페이스에 추가 예정
```

같은 워크스페이스라서 **두 가지가 공유**돼요.

| 공유 항목 | 내용 | 똥쌀때 영향 |
|---|---|---|
| 발송량 한도 | 워크스페이스당 **10만 건** | 똥쌀때는 옵트인 1종뿐이라 거의 안 씀 → 문제 없음 |
| 중복 캠페인 금지 | 동일 마케팅 목적(세그먼트·랜딩·소재)으로 여러 캠페인 = 어뷰징 반려 | 똥쌀때는 마케팅 캠페인을 안 돌리니 충돌 없음. **검수 중 앱과 문구·목적이 겹치는 캠페인만 피하면 됨** |

➡️ **결론: 같은 워크스페이스로 가도 안전해요.** 단, 나중에 두 앱 모두 마케팅 푸시를 돌릴 거면 그때 목적/세그먼트를 분리하세요.

---

## 2. 비용 0원으로 유지하는 법

| 항목 | 0원 방법 | 피해야 할 것 |
|---|---|---|
| 발송 방식 | **'토스에게 발송 요청하기'** (서버 불필요) | '직접 API로 발송' = 파트너 서버 필요 → 서버비 발생 |
| 발송량 | 워크스페이스 **무료 10만 건** 안에서만 | 대량 마케팅 발송 |
| 앱 호스팅 | 정적 빌드(`npm run build`) → 무료 정적 호스팅 | 상시 구동 서버 |
| 알림 빈도 | 하루 1회·옵트인만 | 다회 발송 |

➡️ 똥쌀때 구조상 발송량이 미미해서 **현실적으로 계속 0원**이에요.

## 4. 다음에 할 일 (순서대로)

### ✅ STEP 1 — 검수 중인 앱부터 마무리 (지금)
- 영업일 **2~3일** 소요. 결과 대기.
- 반려되면 사유 반영해서 재요청.
- 이 기간에 똥쌀때 빌드/카피를 다듬어 두기(아래 준비물).

### ⬜ STEP 2 — 똥쌀때를 같은 워크스페이스에 등록
1. 앱인토스 콘솔 → 워크스페이스 선택 → **미니앱 추가**
2. **미니앱 정보**를 정확히 채우기 (← 알림 동의문에 자동 반영됨)
   - 서비스명 / 서비스 설명 / **고객센터 연락처**
3. `npm run build` 결과물을 무료 정적 호스팅에 올리고, 미니앱 URL 연결
4. 실제 접속·동작 확인

### ⬜ STEP 3 — (알림 쓸 경우만) 알림 동의문 등록
> 똥쌀때는 "사용자가 정한 시간에 매일" 보내는 정기 알림이라 **알림 동의문이 필수**예요.
- 콘솔 → 스마트 발송 → **알림 동의문** 탭 → 등록
- 입력값은 `src/content/notifications.ts`의 `NOTIFY_CONSENT` 그대로:
  - 이름: `마음챙김 정기 알림 동의`
  - 발송 시점: `매일 사용자가 설정한 시간에 마음챙김 알림을 보내드려요.`
  - 발송 방법: `정해진 요일과 시간 (매일 1회)`
  - 철회 경로: `똥쌀때 > 설정 > 마음챙김 알림에서 언제든 끌 수 있어요.`

### ⬜ STEP 4 — 캠페인 생성 + 문구 검수 요청
- **기능성 캠페인** 생성 → **'토스에게 발송 요청하기'** → **정기 발송** 선택
- 제목/내용: 아래 5번 표의 기능성 템플릿 사용
- 이동 URL: 똥쌀때 **세션 화면으로 바로 들어가는 딥링크** (접속 확인 필수)
- 문구 검수 **2~3영업일**
- ⚠️ 검수자가 "서비스 유도성"으로 보면 광고성으로 반려될 수 있어요 → 문구를 중립·정보형으로 유지

### ⬜ STEP 5 — 승인 후 발송 시작
- 앱에서 알림을 **켠 사용자만** 대상이 됨
- 워크스페이스 공유 발송량 모니터링

| 항목 | 값 |

| 발송 시점 | 매일 사용자가 설정한 시간에 마음챙김 알림을 보내드려요. |
| 발송 방법 | 정해진 요일과 시간 (매일 1회) |
| 철회 경로 | 똥쌀때 > 설정 > 마음챙김 알림에서 언제든 끌 수 있어요. |
| 서비스명·설명·고객센터 | 미니앱 정보에서 자동 반영 |

### 기능성 정기 알림 (사용자가 켠 경우만) — **권장**
| 제목 (≤7) | 내용 (≤25) | 제목수 | 내용수 |
|---|---|---|---|
| 마음챙김 | 오늘의 마음챙김 시간이에요. | 4 | 15 |
| 오늘 쉼 | 잠시 마음 챙기는 시간이에요. | 4 | 16 |

### (참고) 광고성 — 나중에 마케팅 전환 시에만
> 마케팅 수신 동의 + 세그먼트 필요. 본문에 '토스에서' 포함.

| 제목 (≤7) | 내용 (≤25) | 제목수 | 내용수 |
|---|---|---|---|
| 잠깐 쉬기 | 토스에서 숨 한 번 고르고 가세요. | 5 | 19 |
| 5분 휴식 | 토스에서 잠시 마음을 비워봐요. | 5 | 17 |

## 7. 똥쌀때에서 특히 지킬 원칙 (디톡스 철학)

1. **리텐션 미끼·FOMO 금지** — "놓치면 후회", "안 하면 손해" 류는 제품 철학과 충돌 + 반려 위험.
2. **항상 사용자가 켠 것만** 발송, 끄기는 한 번에.
3. **하루 1회 이하**, 사용자가 정한 시간에만.
4. 문구는 *부르는 말*이 아니라 *조용히 알려주는 말*로.
